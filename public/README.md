# ihelpwithai.com

Static GitHub Pages site for helping non-technical people find the right AI tools in plain English.

## Core idea
This is a curated AI tools site.
It is **not** meant to be the biggest directory.
It is meant to make choosing easier.

## Source of truth
Edit these files:
- `tools-starter.json` — tool data
- `comparisons.json` — comparison page data
- `index.html` — homepage structure and copy
- `styles.css` — site styles
- `app.js` — homepage behavior
- `tool.js` — tool page behavior
- `affiliate-disclosure.html`
- `editorial-methodology.html`
- `best-free-ai-tools.html`

## Generated files
These are generated by the build script and should not be manually edited unless you know exactly why:
- `tools/`
- `guides/`
- `comparisons/`
- `data.js`
- `tools-starter.csv`
- `sitemap.xml`
- `public/`

## Build
```bash
npm run build
```

This generates:
- tool pages
- guide pages
- comparison pages
- `data.js`
- `tools-starter.csv`
- `sitemap.xml`
- deploy-ready `public/` output

## Publish
This repo is for a GitHub Pages website.
If deployment is tied to the built site output, make sure generated files are committed after a successful build.

Recommended workflow:
1. Update content or site code
2. Run `npm run build`
3. Review generated output
4. Commit changes
5. Push to GitHub

## Content rules
Write for people who are not tech savvy.

Every tool should make it easy to understand:
- what it does
- who it is for
- one real use case
- watch-outs
- what to try first

Avoid:
- unnecessary AI jargon
- hype language
- giant uncurated lists
- hiding tradeoffs
